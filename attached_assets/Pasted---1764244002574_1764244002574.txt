أريد منك تطوير النظام الإداري الحالي بإضافة صفحة جديدة داخل النظام باسم:
“المرسلات والكتب”
وتحويلها إلى نظام مرسلات حكومي متكامل (Incoming & Outgoing Documents System)
مبني داخل النظام الموجود حالياً بدون إنشاء مشروع جديد.

الرجاء تنفيذ المتطلبات التالية بالتفصيل:

========================================================
## 1) إنشاء صفحة “المرسلات والكتب”
داخل النظام الإداري، أضف قسم جديد بالقائمة الجانبية باسم:
"المرسلات والكتب"

وتحتها 3 صفحات رئيسية:
- قائمة الكتب والمرسلات (Document List)
- إضافة كتاب/مراسلة جديدة (Add Document)
- استيراد كتاب من الجهاز (Import Document)

جميع الصفحات يجب أن تكون بنفس التصميم الموجود حالياً (Blade + Tailwind + RTL).

========================================================
## 2) إضافة مراسلة جديدة (Add Document)
إنشاء نموذج إضافة مراسلة يحتوي على الحقول:
- رقم الكتاب أو المراسلة (Document Number)
- عنوان الكتاب
- نوع المراسلة (وارد / صادر)
- الجهة المرسلة أو المستلمة
- تاريخ المراسلة
- الموضوع (Subject)
- وصف مختصر
- الحالة (جديد / تمت المراجعة / مكتمل)
- رفع ملف الكتاب (File Upload: Word/PDF/Images/Any)

بعد الحفظ:
- تخزين الملف في مجلد document_storage
- تخزين بيانات الكتاب داخل قاعدة البيانات

========================================================
## 3) استيراد كتاب من الجهاز (Import Document)
إنشاء صفحة استيراد تدعم رفع الملفات التالية:
- Word: .doc / .docx
- PDF
- صور: jpg / png / webp
- أي صيغة أخرى

بعد رفع الملف:
- إذا الملف Word:
    • تحقق من وجود Microsoft Word في النظام
    • إذا Word غير مثبت → أظهر رسالة:
      "يجب تثبيت برنامج Word لفتح وتعديل هذا النوع من الملفات"
    • إذا كان مثبتاً → توفير زر لفتح الملف داخل Word للتعديل
- إذا PDF:
    • عرض معاينة داخل النظام
- إذا صورة:
    • عرضها مباشرة داخل النظام
- أي صيغة أخرى → حفظها كملف مرفق للعرض أو التحميل

========================================================
## 4) دمج Word مع النظام (Office Integration)
أريد دعم التكامل مع Microsoft Word كالتالي:

- إذا المستخدم فتح ملف Word:
    • يتم محاولة فتحه عبر بروتوكول:
        ms-word:ofe|u|<file_path>
    • إذا النظام لا يدعم بروتوكول Word → إظهار تنبيه
    • توفير زر “فتح الملف في Word”

- عند حفظ الملف في Word، يجب أن يعود إلى النظام ويتم تحديثه.

========================================================
## 5) عرض المستندات داخل النظام (Document Viewer)
أريد صفحة عرض كتاب (View Document) تستعرض الملف حسب نوعه:

### إذا كان PDF:
عرضه داخل iframe بشكل مباشر.

### إذا كان Word:
- إذا Word موجود → زر “فتح في Word”
- إذا Word غير موجود → رسالة:
  “لا يمكن فتح هذا المستند – يجب تثبيت Microsoft Word”

### إذا كان صورة:
عرضها مباشرة داخل الصفحة.

### إذا كان أي صيغة أخرى:
عرض زر تحميل فقط.

========================================================
## 6) قاعدة البيانات المطلوبة
إنشاء جدول جديد داخل المشروع الحالي باسم documents:

documents:
- id
- document_number
- title
- type (incoming/outgoing)
- from_department
- to_department
- date
- subject
- description
- file_path
- file_type
- status
- created_by
- updated_at

========================================================
## 7) البحث في المرسلات
إنشاء صفحة بحث تسمح بالبحث بواسطة:
- رقم الكتاب
- العنوان
- نوع المراسلة
- الجهة
- التاريخ
- نوع الملف

========================================================
## 8) دمج الوحدة داخل النظام الحالي
- إضافة روابط للوحدة في الـ Sidebar
- استخدام نفس الـ layout العام للنظام
- استخدام نفس الألوان والهوية البصرية
- عدم إنشاء أي مشروع جديد – فقط إضافة الموديل والمهاجرات والصفحات داخل النظام.

========================================================
## 9) مخرجات التطوير
أريد منك إنشاء التالي بالكامل:
- Controller كامل لإدارة الكتب (DocumentsController)
- Model documents
- Migration كاملة
- صفحات Blade:
    • documents/index.blade.php
    • documents/create.blade.php
    • documents/import.blade.php
    • documents/view.blade.php
- دعم عرض الملفات حسب نوعها
- دعم التكامل مع Word وPDF
- دعم الاستيراد والرفع
- بحث متقدم
- زر فتح في Word
- إظهار رسالة عند عدم وجود Word

ابدأ الآن في تطوير هذا الجزء داخل المشروع الحالي فقط.
