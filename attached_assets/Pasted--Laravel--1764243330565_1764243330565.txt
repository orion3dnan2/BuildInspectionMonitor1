أريد منك إنشاء نظام صلاحيات كامل وشامل داخل مشروع Laravel الخاص بي،  
يكون مركز الصلاحيات الرئيسي (Permissions Center) ويغطي كل جزء من النظام  
بدون استثناء، بما في ذلك: كل الصفحات، كل التبويبات، كل الأقسام، وكل الأزرار الحساسة.

الرجاء تنفيذ التالي بدقة:

-----------------------------------------------------------
## 1) نظام الصلاحيات العام (Roles & Permissions)
إنشاء 3 مستويات:
- مدير النظام (Admin)
- مشرف (Supervisor)
- مستخدم (User)

وتكون لكل مستوى صلاحيات محددة من خلال Permission System  
وليس فقط Role Based.

-----------------------------------------------------------
## 2) بناء جدول permissions + roles + role_permissions + user_roles
إنشاء جداول:

permissions:
  id, name, key, description

roles:
  id, name, description

role_permissions:
  role_id, permission_id

user_roles:
  user_id, role_id

استخدام Model + Migration كاملة.

-----------------------------------------------------------
## 3) إنشاء لوحة إدارة الصلاحيات (Permissions Management UI)
داخل صفحة "المستخدمين" أريد صفحة جديدة:
**قسم إدارة الصلاحيات**
تعرض:

- قائمة بكل صفحات النظام
- كل التبويبات في الواجهة
- كل وحدات النظام البلوكات + النظام الإداري
- كل الأزرار المهمة (إضافة – تعديل – حذف – اعتماد – استيراد – تصدير – كتابة – إرسال – تدقيق… إلخ)

التصميم المطلوب:
- جدول كبير يحتوي كل صفحة
- عمود "قراءة"، "إضافة"، "تعديل"، "حذف"، "اعتماد"، "استيراد"، "عرض الإحصائيات"
- Switch button أو CheckBox لكل صلاحية

عند حفظ الصلاحيات:
يتم تحديث role_permissions تلقائياً.

-----------------------------------------------------------
## 4) استخراج كل الصفحات من المشروع تلقائياً
أريد منك مسح كامل المشروع واستخراج كل الصفحات:
- لوحة التحكم Dashboard
- إدخال البيانات
- البحث والاستعلام
- التقارير والإحصائيات
- الاستيراد
- الإعدادات
- إدارة المستخدمين
- إدارة المنافذ
- إدارة المخافر
- نظام البلوكات (جميع صفحاته)
- النظام الإداري (جميع الصفحات)
- أي صفحة فرعية أخرى

كل صفحة يجب تسجيل Permission لها تلقائياً بصيغة:
page.view, page.create, page.update, page.delete, page.approve, page.import

-----------------------------------------------------------
## 5) حماية كل صفحة بواسطة Middleware + Gate
إنشاء:
PermissionMiddleware
الذي يفحص:
- هل المستخدم لديه Role مناسب؟
- هل يمتلك Permission المطلوب لهذه الصفحة؟

إذا ما عنده صلاحية:
يتم تحويله لصفحة "غير مسموح لك"

-----------------------------------------------------------
## 6) تعديل كل الروابط الفرعية بحيث تختفي إن لم يمتلك المستخدم الصلاحية
داخل Sidebar وNavbar:
- إذا لا يملك إذن page.view → أخفي الرابط
- إذا لا يملك إذن page.create → أخفي زر إضافة
- وكذلك باقي الأزرار

-----------------------------------------------------------
## 7) واجهة المستخدم
أريد واجهة Blade + Tailwind:
- صفحة إدارة الصلاحيات
- صفحة عرض الصلاحيات للمستخدم
- صفحة تعديل الصلاحيات
- صفحة “غير مسموح لك”

-----------------------------------------------------------
## 8) مخرجات مهمة:
يجب إنشاء:

✓ Migrations كاملة  
✓ Models كاملة  
✓ Controllers كاملة  
✓ Middleware  
✓ Gates  
✓ Views  
✓ تعديل جميع الصفحات لقراءة الصلاحيات  
✓ Sidebar ديناميكي حسب الصلاحيات  
✓ زر صلاحيات داخل صفحة المستخدم (manage permissions)

-----------------------------------------------------------

## 9) المطلوب النهائي
نظام صلاحيات حكومي متكامل وشامل،  
يغطي كل جزء وكل صفحة وكل وظيفة داخل المشروع  
بدون نسيان أي جزء.

ابدأ الآن في تنفيذ جميع الخطوات بالكامل.
